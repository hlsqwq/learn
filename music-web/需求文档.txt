## 1. 项目视觉与设计语言 (Design Language)

为了打破同质化，建议采用 **"Glassmorphism"（玻璃拟态）** 与 **"Dynamic Adaptive Color"（动态自适应色彩）** 的结合。

* **沉浸式背景：** 页面背景不应是纯色，而是根据当前播放歌曲的封面，通过高斯模糊算法提取出 2-3 个主色调，形成流动的弥散光背景。
* **非线性布局：** 摆脱传统的左侧固定导航栏，尝试使用底部浮动 Dock 栏或隐藏式侧边栏。
* **微交互 (Micro-interactions)：** 播放按钮的形态变化、封面图在播放时的呼吸律动感、进度条拖动时的阻尼感。
* **字体排版：** 使用无衬线字体（如 *Inter* 或 *PingFang SC*），拉大行间距，利用字号大小对比而非粗细对比来区分层级。

## 2. 核心功能模块 (Core Features)

### 2.1 发现 (Discovery) - 沉浸式首页
* **动态头图：** 以大面积横向卡片展示推荐歌单，鼠标悬停时卡片微弱放大并伴随光影变换。
* **个性化雷达：** 基于标签（而非简单的分类）生成放射状或蜂巢状的音乐流。
* **时段推荐：** 根据时间（清晨、深夜、工作时）自动切换 UI 主题色和推荐曲风。

### 2.2 播放控制 (Playback Engine)
* **无缝衔接 (Gapless Playback)：** 歌曲切换时音量淡入淡出。
* **可视化频谱：** 提供 2-3 种极简的频谱动效（如线条波动、像素点阵），增加视听一体感。
* **歌词实验室：** 借鉴 Apple Music 的逐字随动歌词，支持“海报模式”，允许用户选定歌词片段生成极简风的分享卡片。

### 2.3 音乐库管理 (Library)
* **智能分类：** 除了歌手/专辑，增加“心情”和“色彩”分类（如：忧郁的蓝色、活力的橙色）。
* **磁贴式收藏：** 收藏动作伴随粒子动效，让“拥有”这首歌的过程更有仪式感。

## 3. 技术架构 (Technical Stack)

| 维度 | 技术栈建议 | 理由 |
| :--- | :--- | :--- |
| **前端框架** | **Vue 3 (Composition API)** | 响应式性能强，易于维护复杂的状态。 |
| **状态管理** | **Pinia** | 管理播放状态、用户信息及全局音量。 |
| **样式处理** | **Tailwind CSS + Framer Motion (Vue)** | 快速实现复杂布局与平滑的过渡动画。 |
| **颜色提取** | **ColorThief / Fast-Average-Color** | 实现封面颜色提取，驱动 UI 背景变换。 |
| **构建工具** | **Vite** | 极速的热更新体验。 |

## 4. 关键交互流程设计 (Interaction Design)

### 4.1 封面提取色流程
当用户切换歌曲时：
1.  前端获取歌曲 URL 及封面图。
2.  `Fast-Average-Color` 库计算封面主色及辅助色。
3.  通过 CSS 变量绑定到根节点：`--theme-primary: #xxxxxx`。
4.  背景组件利用 `backdrop-filter: blur(80px)` 实现色彩的柔和扩散。

### 4.2 极简搜索体验
* 按下 `Cmd+K` (或 `Ctrl+K`) 唤起全屏搜索蒙层，而非传统的搜索框。
* 搜索结果随输入实时流式渲染（Debounce 处理），优先展示匹配度最高的封面。

## 5. 阶段性路线图 (Roadmap)

1.  **Phase 1 (MVP):** 搭建 Vue 3 骨架，集成基础播放功能（Play/Pause/Skip），实现响应式基础布局。
2.  **Phase 2 (Visual):** 引入封面颜色提取算法，完成高斯模糊动态背景，添加页面切换动画。
3.  **Phase 3 (Experience):** 开发逐字歌词组件、音量淡入淡出逻辑、搜索悬浮窗。
4.  **Phase 4 (Refine):** 性能优化（图片懒加载、组件异步加载）、个性化主题定制。
